# しほしけコンバーター（β版）

司法試験論文式の「試験問題」「出題の趣旨」「採点実感」を、法務省ウェブサイトで
公表されているPDFから自動で抽出してテキスト化する、法務省非公式のWebアプリ。
処理はすべてブラウザ内で行われ、サーバー側にデータは保存されない。

公開ページ: https://adatchan.github.io/barexam-scrapbox-web/

姉妹ページとして、短答式の問題と正答及び配点を年度別・科目別に保存する
**しほしけ短答ダウンローダー**（`tantou.html`）を同梱する。
公開ページ: https://adatchan.github.io/barexam-scrapbox-web/tantou.html

両ページとも、上部の「試験」ラジオで**司法試験 / 予備試験**を切り替えられる。
予備試験は科目グループ別（憲法・行政法 など）にPDFが公表され、論文式の出題の趣旨は
全科目をまとめた1PDF・採点実感は無い。予備試験論文式は、司法試験と同じ
**科目別の全文テキスト変換（ノーマル／Scrapbox記法）**と、**出典フッター・
見出し付きの原典PDF保存**（短答=問題＋正答及び配点、論文=試験問題＋出題の趣旨）の
両方に対応する。テキスト変換は `yobi-convert.js` が本文の科目見出し（［憲法］ 等）を
境界に当該科目だけを切り出して司法と同じ段落復元・整形に流し、PDF保存は
`pdfsplit.js` がページ先頭の科目見出しからページ範囲を求めて抜粋する。
画像化された一部年度の趣旨はテキスト抽出できないため、テキスト変換はエラーで
PDF保存を案内し、PDF保存では全体を保存する。短答式の正答及び配点は
1ページの表のためグループ単位で保存する。

## 機能

- **テキスト変換**: 司法試験は年度（平成22年〜最新）× 科目（基本7科目＋経済法・
  労働法・倒産法 各第1問/第2問）× 種類、予備試験は年度（平成23年〜最新）×
  科目（基本7科目＋法律実務基礎科目（民事・刑事）＋選択8科目）×
  種類（試験問題・出題の趣旨）を選んで変換。出力形式はノーマル（プレーン）と
  Scrapbox記法（`[* ]` 見出し・`#タグ`付き）を選択可
- **原典PDF保存**: 原典PDFから該当ページだけを抜き出し、各ページのフッターに
  ファイル名と出典・加工表示を印字したPDFを保存
- **一式zip保存**: 試験問題・出題の趣旨・採点実感の3点（予備は問題・趣旨の2点）の
  抜粋PDFを「[年度]…[科目]一式」フォルダにまとめてzipで保存
- **LLM用に保存**: 試験問題・出題の趣旨・採点実感（予備は問題・趣旨）を、
  メタ情報・出典付きの1つのMarkdownにまとめて保存（LLMに添付して答案の
  添削・解説を依頼する用）
- **処理結果キャッシュ**: 同じ年度・科目・種類の再処理を省略（件数・容量で上限管理）
- **新年度の自動対応**: 毎週月曜にGitHub Actionsが法務省サイトを巡回し、新年度の
  掲載を検出すると年度リストと「更新情報」欄を自動更新してデプロイ
- **出典表示**: 出力には公共データ利用規約（PDL1.0）に基づく
  「出典：法務省ウェブサイト（URL）を加工して作成」を自動付与

## しほしけ短答ダウンローダー（tantou.html）

司法試験「短答式」の問題と「正答及び配点」を、年度別・科目別に保存する姉妹ページ。
論文式と異なり、短答式には出題の趣旨・採点実感が無く、科目は憲法・民法・刑法の3科目
（この3科目別の出題になった平成27年以降が対象）。短答式は選択式でテキスト化に
適さないため、テキスト変換は行わず原典PDFの保存に特化している。

- **対応**: 平成27年〜最新年度 × 憲法・民法・刑法。問題・正答及び配点を保存
- **保存**: 問題PDF単体 / 正答・配点PDF単体 / 両者をまとめた zip。いずれも全ページに
  出典・加工表示のフッターを印字（pdfout.js を論文式と共有）
- **リンク探索**: 試験問題ページの《短答式試験》区分から問題PDFを、結果ページ →
  「短答式試験」サブページ →《正答及び配点》（年度により「正解及び配点」表記）から
  正答PDFを特定。探索本体は DOM 非依存の `tantou-moj.js` に置き、画面（`tantou.js`）と
  Node の巡回・検査スクリプトが同じコードパスを共有する
- **新年度の自動対応**: 週次クロール（`update-years.mjs`）が短答式問題・正答及び配点の
  掲載を検出し、年度リストと短答専用の「更新情報」（`news.js` の `TANTOU_NEWS`）を
  自動更新。掲載確認の記録は `TANTOU_CRAWL_STATE`（年度ごとに 問題=mondai・
  正答及び配点=seikai）。リンク健全性チェック（`check-links.mjs`）も短答の問題・正答を検査する

## 構成

```
web/
├── index.html              論文式コンバーターの UI マークアップ
├── tantou.html             短答ダウンローダーの UI マークアップ
├── tantou.js               短答ダウンローダーの UI 層（取得・PDF保存の制御）
├── tantou-moj.js           短答式のリンク探索（DOM非依存・画面/巡回で共有）
├── yobi-years.js           予備試験の年度→法務省ページURL対応表（自動生成）
├── yobi-moj.js             予備試験のリンク探索・科目定義（DOM非依存・共有）
├── pdfsplit.js             PDFの科目別ページ抽出・先頭ページ判定（PDF.js・共有）
├── style.css               スタイル（両ページ共有）
├── app.js                  UI 層（イベントハンドラ・表示制御）
├── rules.js                テキスト構造の共有ルール（正規表現・判定）
├── data.js                 科目定義などのデータテーブル
├── years.js                年度→法務省ページURL対応表（自動生成）
├── news.js                 更新情報と巡回状態（自動生成）
├── moj.js                  法務省ウェブからの取得（Worker 中継・リンク探索）
├── parser.js               PDF解析（PDF.js）と段落構造の復元
├── format.js               テキスト整形（ノーマル / Scrapbox 記法）
├── convert.js              変換ディスパッチと処理結果キャッシュ
├── yobi-convert.js         予備試験論文式のテキスト変換（科目グループPDFから切り出し）
├── pdfout.js               原典PDFの抜き出し・スタンプ印字・zip
├── worker/worker.js        Cloudflare Worker（moj.go.jp 中継プロキシ）
├── scripts/
│   ├── update-years.mjs    法務省サイト巡回 → years.js / news.js 更新
│   ├── check-links.mjs     全年度×全種類のリンク探索の健全性チェック
│   └── bump-version.mjs    index.html のキャッシュバスター（?v=）更新
├── .githooks/pre-commit    JS/CSS 変更時にキャッシュバスターを自動更新
└── .github/workflows/
    └── update-years.yml    週次の自動巡回・健全性チェック（毎週月曜 09:00 JST）
```

- **フロント**: 静的 HTML/CSS/JS（ビルド不要）。GitHub Pages で配信
- **バックエンド**: なし。法務省サイトの取得だけ Cloudflare Worker が中継
  （moj.go.jp 配下のみ許可・CORSヘッダ付与）
- **PDF**: 解析は PDF.js、抜粋PDFの生成は pdf-lib、zip は fflate
  （いずれも CDN から動的ロード）
- **セキュリティ**: 両HTMLに CSP メタタグを設定し、スクリプトの読み込み先を
  自サイト＋cdnjs＋jsDelivr に限定（インラインスクリプトは sha256 ハッシュで
  個別許可）。index.html の pdf.js 静的タグには SRI（integrity 属性）を付与

## 開発メモ

クローンしたら一度だけ、コミットフックを有効化する:

```sh
git config core.hooksPath .githooks
```

これにより JS/CSS を変更してコミットすると index.html の `?v=` が自動更新される
（GitHub Pages の最大10分キャッシュ対策）。手動更新は
`node scripts/bump-version.mjs`。

リンク探索の健全性チェック（全年度×全種類、約2分）:

```sh
node scripts/check-links.mjs
```

法務省サイトの巡回（新年度の検出・更新情報の追記）:

```sh
node scripts/update-years.mjs
```

いずれも週次ワークフロー（update-years.yml）が自動実行する。失敗すると
GitHub からワークフロー失敗が通知される。
